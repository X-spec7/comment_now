# リソース要件

## サーバー要件

現在は1台のUbuntu VPSサーバーで十分です。

## AWS EC2インスタンスの作成

EC2インスタンスを作成する際に、SSHキーを登録し、以下の情報が必要です：

- SSHキーペア
- インスタンスのIPアドレス（サーバーアクセス用）

### セキュリティグループ設定（インバウンドルール）

IAMユーザーアクセスを提供しない場合は、以下のインバウンドルールを設定してください：

- **SSH (Port 22)**: サーバーアクセス用のあなたのIPアドレス
- **HTTP (Port 80)**: 0.0.0.0/0（Webアクセス用）
- **HTTPS (Port 443)**: 0.0.0.0/0（セキュアWebアクセス用）
- **Custom TCP (Port 3000)**: 0.0.0.0/0（フロントエンド開発用）
- **Custom TCP (Port 8000)**: 0.0.0.0/0（バックエンドAPI用）

## 代替案：IAMユーザー

または、AWSアカウントでIAMユーザーを作成し、提供していただければ、すべての設定をこちらで行うことができます。

### 必要なIAMユーザー権限

IAMユーザーには以下のポリシーをアタッチしてください：

- **EC2FullAccess**: EC2インスタンス管理用
- **DynamoDBFullAccess**: データベース操作用
- **S3FullAccess**: ファイルストレージ用（必要に応じて）
- **IAMReadOnlyAccess**: IAMポリシー読み取り用
- **CloudWatchLogsFullAccess**: アプリケーションログ用

## メールサービス

SendGridなどのメールサービスを提供していただければ、メール認証機能の統合も可能です。

ただし、開発段階ではコストを考慮してメール認証をバイパスし、後で統合することを推奨します。
